## Issues

### P2 ‚Äî Nested block structure inside `blockquote` / `pre`

`convertBlockquote` always emits **one** `Paragraph` and runs children through **inline** collection.  
`<blockquote><p>‚Ä¶</p><p>‚Ä¶</p></blockquote>` flattens via `convertInline` on the `p` elements (works for simple text) but is not true multi-paragraph quote structure.

**Advice:** Accept for Stage 3. Optional Stage 8: if blockquote children are block-level, recurse `convertBlock` and apply indent/border to each resulting paragraph.

### P2 ‚Äî Non-container unknown tags drop content

Direct body children that are neither block handlers nor `BLOCK_CONTAINER_TAGS` return `[]` ‚Äî e.g. stray `<span>‚Ä¶</span>`, bare `<a>`, custom elements. Content vanishes with no warning.

**Advice:** Stage 8 or small hardening: unknown phrasing tags ‚Üí wrap in `Paragraph` via `convertInline`; log once in verbose mode. Not required before lists/tables.

### P2 ‚Äî Lists/tables silently omit body text today

Confirmed: `<p>before</p><ul><li>item</li></ul><p>after</p>` ‚Üí docx text **‚Äúbefore after‚Äù** only. Documented intentional; **HANDOFF is correct**. Stage 4/5 owners must not treat ‚Äúcontent missing‚Äù as a regression until those stages land ‚Äî but a one-line README ‚Äúknown gap‚Äù helps humans.

### P3 ‚Äî `h2`‚Äì`h6` under-tested

Fixture includes `h2`; assertions only lock **Heading1** + CSS on h1. Low risk (same `HEADING_LEVELS` map). Optional: one assert on `Heading2`.

### P3 ‚Äî Verify script still not host-complete

Visual used one-off Windows LO; `scripts/verify.sh` / `soffice.py` still assume PATH `soffice`; `pdftoppm` still missing. Does **not** reopen Stage 3 if visual was done manually ‚Äî assign `[janus]` for VERIFY wrapper when Marcus wants it.

### P3 ‚Äî `figure` / `figcaption` as containers

Flattening is fine; captions become normal paragraphs (no Caption style). OK until media stage.
