## Issues (non-blocking)

### P2 — Verify soft-fail means “OK” without PDF
If LO is missing, conversion still exits 0 with a warning. Fine for a general CLI; **stage DoD** must still use hard PDF path (`scripts/verify.sh` or assert PDF exists). Don’t treat CLI default alone as layout DoD.

### P2 — `scripts/verify.sh` → CLI with verify on
Double LibreOffice work when `verify.sh` calls `bin/cli.js` (CLI verifies, then script converts again). Harmless waste; later point `verify.sh` at `verify.js` only after a single convert.

### P2 — Batch default verify is heavy
Directory mode without `--no-verify` runs LO per file. Document in help (optional one-liner) or default batch to `--no-verify` with explicit `--verify` — product taste, Stage 8.

### P2 — ODT is pandoc round-trip
Loses some native docx fidelity (expected). Errors if pandoc missing — good.

### P3 — Help says “always” verify; implementation is soft-fail
Help text is accurate enough (“PDF check”); BUILD_PLAN explains soft-fail well.

### P3 — No Janus accept line was in HANDOFF for Stage 7 before this review
Process: HANDOFF already flagged “Stage 7 not yet reviewed” — correct.
